# 🚀 LeetCode Daily Auto-Fetch (力扣每日一题自动抓取)

> 🤖 **声明**：本项目的核心自动化脚本、GitHub Actions 配置及本文档均由 AI 辅助编写与优化。

本项目是一个基于 Python 和 GitHub Actions 构建的全自动 LeetCode（力扣中国区）每日一题抓取与归档工具。它能够每天定时在云端运行，自动获取当天的题目，将其保存为多种离线格式，并为你准备好专属的 Markdown 刷题笔记模板。

---

## ✨ 项目功能

本项目旨在为你提供“零干预”的沉浸式刷题体验，具有以下核心特性：

* **🤖 全自动化运行**：基于 GitHub Actions，每天北京时间 00:00 自动触发，无需本地服务器，零成本运行。
* **📄 多维度题目归档**：使用 Playwright 渲染网页，自动将题目详情保存为高质量的 **PDF 文件** 和 **全网页截图 (PNG)**，并配置了专门的中文字体（Noto CJK）解决乱码问题。
* **📝 智能笔记模板**：自动生成当天的 `.md` 笔记文件，内含题目链接、难度标注以及 C++ 代码块和时空复杂度分析占位符。
* **📂 结构化目录存储**：按 `年 / 月 / 年-月-日` 的清晰层级自动创建文件夹，彻底告别文件杂乱。
* **📊 README 自动维护**：脚本会自动读取并重构仓库的 README 文件，按年月动态生成题目索引表，最新题目自动置顶。

---

## 🛠️ 技术栈

* **语言**: Python 3.11
* **核心依赖**: `requests`, `playwright`
* **自动化**: GitHub Actions (`ubuntu-latest`)

## 🚀 使用方法 (Fork 本项目)

如果你想在自己的 GitHub 上使用这套自动化工作流，请按照以下步骤操作：

### 1. Fork 本仓库
点击本页面右上角的 **Fork** 按钮，将项目复制到你自己的 GitHub 账号下。

### 2. 开启 GitHub Actions 读写权限 (⚠️ 核心步骤)
由于脚本需要向你的仓库提交新生成的文件，你必须赋予 GitHub Actions 写入权限：
1. 进入你 Fork 后的仓库，点击顶部的 **Settings**。
2. 在左侧边栏找到 **Actions** -> **General**。
3. 滚动到底部找到 **Workflow permissions**，选中 **Read and write permissions**。
4. 点击 **Save** 保存。

### 3. 手动触发首次运行 (可选)
工作流默认会在每天北京时间 00:00 自动运行，但你可以随时手动测试：
1. 点击仓库顶部的 **Actions** 标签页。
2. 如果提示 "I understand my workflows, go ahead and enable them"，请点击确认。
3. 在左侧点击 **LeetCode Daily Auto-Fetch**。
4. 点击右侧的 **Run workflow** 下拉菜单，然后点击绿色的 **Run workflow** 按钮。

### 4. 每日打卡
一切配置妥当后，你只需：
1. 每天将仓库 `git pull` 到本地。
2. 在当天生成的 `.md` 文件中编写你的代码和题解。
3. 将你的题解 `git push` 到云端，完成当天的打卡！
